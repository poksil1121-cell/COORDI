# 게이지 툴팁 & 근거 기반 제품 추천 설명 — 설계

날짜: 2026-07-09

## 배경

- 날씨 패널의 습도/바람/강수/자외선 게이지가 숫자만 보여주고 그 의미를 설명하지 않는다.
- 헤어/스킨케어 카드의 조언(특히 "신규 제품 제안")이 왜 그 제품 종류가 필요한지 근거를 설명하지 않는다.

## 사전 확인 (요청받은 점검)

지도로 지역을 바꾼 뒤 헤어/스킨케어/코디 추천이 새 지역 날씨에 맞게 갱신되는지 실제 브라우저(Playwright)로 확인함: 제주 → 강풍 지역으로 변경 시 습도(78%→94%), 코디 조언("민소매" → "반팔" + 방풍 아우터 언급), 헤어 조언(강풍 문구 추가)이 모두 정확히 갱신됨. 별도 수정 불필요 (지난 버그 수정으로 이미 해결됨).

## 신뢰성 관련 제약 (중요)

"의학논문/저널 기반"이라는 표현대로 실제 특정 논문을 인용하는 건 하지 않는다 — 존재하지 않는 논문을 지어내면 신뢰도를 높이려다 사용자를 속이는 결과가 된다. 대신:

- 출처는 **실제로 존재하고 공개적으로 검증 가능한 기관의 일반 가이드라인**(예: 미국피부과학회 AAD, 세계보건기구 WHO의 자외선 지수 등급)만 인용
- 특정 논문으로 뒷받침하기 애매한 내용은 "일반적으로 알려진 피부/모발 과학 원리"로 표현하고 특정 출처를 지어내지 않음
- 항상 "일반 정보이며 개인차가 있을 수 있고, 지속되는 증상은 전문가와 상담 권장" 문구를 표시 (기존 `server.js`의 AI 코멘트 시스템 프롬프트에 있는 안전장치와 같은 원칙)

## 1. 게이지 툴팁

습도/바람/강수/자외선 게이지에 마우스를 올리면(또는 포커스하면) 그 수치가 무엇을 의미하는지 보여주는 순수 CSS 툴팁을 추가한다. 내용은 실제로 앱 내부 로직이 사용하는 기준값을 그대로 설명해서 앱 동작과 100% 일치시킨다:

| 게이지 | 툴팁 내용 |
|---|---|
| 습도 | "공기 중 수분 비율이에요. 65% 이상이면 습함, 35% 이하면 건조로 구분해서 추천에 반영해요. 습하면 모발 컬이 풀리기 쉽고, 건조하면 피부·모발 수분이 빠르게 손실돼요." |
| 바람 | "시속 풍속(km/h)이에요. 20km/h 이상이면 강풍으로 분류해서 헤어 고정과 아우터를 더 신경 쓰도록 추천해요." |
| 강수 | "오늘 비가 올 확률이에요. 50% 이상이면 우산·방수 신발을 챙기도록 추천해요." |
| 자외선 | "자외선 지수(UV Index)예요. WHO 기준 0~2 낮음, 3~5 보통, 6~7 높음, 8~10 매우높음, 11+ 위험 등급이며, 6 이상부터 선크림 재도포를 추천해요." |

- 데스크톱: 마우스 호버 시 표시
- 키보드/모바일 접근성: 게이지에 `tabindex="0"`을 줘서 포커스로도 표시 가능
- 게이지 위(공간이 부족하면 아래)에 작은 말풍선 형태로 표시

## 2. 근거 기반 제품 추천 설명

헤어 스타일링·스킨케어 카드의 각 조언(보유 제품 활용/신규 제품 제안 모두 포함)에 "💡 전문가 팁" 트리거를 추가한다.

- **호버 시**: 한 줄 요약(예: "습도가 높으면 모발 큐티클이 부풀어 컬이 쉽게 풀려요")을 툴팁으로 표시 (1번과 같은 CSS 툴팁 방식 재사용)
- **클릭 시**: 화면 중앙에 모달이 열리며 다음을 보여줌:
  - 요약
  - **근거**: 왜 그 상황에서 그 제품 종류가 도움이 되는지 설명하는 일반적인 피부/모발 과학 원리
  - **출처**: 실제 기관의 공개 가이드라인(있는 경우) 또는 "일반적으로 알려진 ○○과학 원리"
  - 항상 표시되는 안내 문구: "이 설명은 일반적인 정보이며 개인 피부·모발 상태에 따라 다를 수 있어요. 증상이 지속되면 전문가와 상담해주세요."

### 대상 및 내용 (헤어/스킨케어 조언 6가지 상황)

| 상황 | 요약 | 출처 |
|---|---|---|
| 습한 날 + 곱슬/웨이브 모발 | 습도가 높으면 모발 큐티클이 부풀어 컬이 쉽게 풀림 | 일반적으로 알려진 모발과학(트라이콜로지) 원리 |
| 습한 날 + 직모 | 습도가 높으면 모발이 붕뜨고 부스스해지기 쉬움 | 일반적으로 알려진 모발과학 원리 |
| 건조한 날 (모발) | 건조한 공기가 모발 수분을 빼앗아 정전기·갈라짐 유발 | 일반적으로 알려진 모발과학 원리 |
| 건조한 날 (피부) | 건조한 환경에서 피부 장벽의 수분이 빠르게 증발 | 미국피부과학회(AAD)의 일반 보습 관리 권고와 일치 |
| 자외선 강한 날 (피부) | 자외선 차단이 피부 노화·손상 예방의 핵심 | 미국피부과학회(AAD)·세계보건기구(WHO) 공통 권고 |
| 고온다습 (피부) | 피지 분비가 늘어 유분·번들거림이 쉽게 생김 | 일반적으로 알려진 피부생리학(피지선 기능) 원리 |

메이크업 카드와 오늘의 코디 카드는 이번 범위에서 제외 (요청에 헤어/스킨케어만 명시됨).

### 구현 방식

- `public/app.js`의 `buildHairRec`/`buildSkinRecs`가 반환하는 각 tip 객체에 `evidence: { summary, rationale, source }` 필드를 상황에 맞게 추가
- 렌더링 시(`tipBlockHtml`, 헤어카드 전용 렌더링) evidence가 있으면 "💡 전문가 팁" 버튼을 추가하고, 클릭 시 공용 모달(`#evidenceModal`, 기존 `#mapModal`과 같은 오버레이 패턴 재사용)에 내용을 채워 보여줌
- 렌더링마다 evidence 데이터를 배열에 등록하고 인덱스만 DOM에 심어서(클릭 시 배열에서 조회), JSON을 HTML 속성에 직접 넣지 않음 (이스케이프 문제 방지)

## 영향 파일

- `public/app.js`: 게이지 렌더링에 툴팁 데이터 추가, `buildHairRec`/`buildSkinRecs`/`tipBlockHtml`/`renderRecommendationCards`에 evidence 연동, 모달 열기/닫기 로직
- `public/index.html`: 헤어카드에 evidence 트리거 요소 추가, `#evidenceModal` 마크업 추가
- `public/style.css`: CSS 툴팁(게이지·evidence 트리거 공용), evidence 모달 콘텐츠 스타일

## 비범위 (Out of scope)

- 실제 의학 논문 인용 (신뢰성 문제로 제외, 공개 기관 가이드라인 또는 일반 원리로 대체)
- 메이크업/코디 카드에 근거 설명 추가
- 게이지 툴팁/근거 설명의 다국어 지원
