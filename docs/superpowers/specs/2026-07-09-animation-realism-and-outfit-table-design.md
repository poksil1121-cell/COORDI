# 날씨 애니메이션 현실감 개선 & 코디 조합표/추천 스타일 & UI 정리 — 설계

날짜: 2026-07-09

## 배경

- 지도 탐색 모달을 다시 열면 이전에 열려있던 위치 팝업이 그대로 남아있는 버그 발견 → 수정 완료 (`closeMapExplorer`가 `exploreMap.closePopup()`을 호출하지 않던 문제).
- 비 애니메이션은 괜찮지만 우박/눈/황사·미세먼지/안개/흐림/번개/폭풍/햇빛 애니메이션이 부자연스럽다.
- 헤어스타일링 카드의 "참고" 배지가 불필요하다.
- 스킨케어의 "신규 제품 제안"이 아직 실제 제품 연동 전이라는 걸 알려야 한다.
- "오늘의 코디" 추천이 한 문장짜리라 너무 단순하다 — 상의/하의/양말/신발까지 구체적으로 보여주는 조합표와, 나중에 핀터레스트류 이미지 연동을 위한 "추천 스타일" 자리가 필요하다.

## 1. UI 정리 (빠른 수정)

- `tagLabel`이 "참고"를 반환하는 경우(헤어 카드의 `tag === "neutral"`), 배지 자체를 렌더링하지 않음
- `tipBlockHtml(tip, showComingSoonHint)`로 시그니처 확장 — 스킨케어 카드 호출(`showComingSoonHint = true`)에서만 `tag === "suggested"`인 배지에 `tooltip-trigger` 클래스와 "아직 실제 제품 추천으로 연동되기 전이에요. 준비 중인 기능이에요." 툴팁을 붙임. 메이크업 카드 호출(`showComingSoonHint = false`)은 기존 그대로.

## 2. 날씨 애니메이션 현실감 개선

비 애니메이션(`raindrop`)은 그대로 둔다. 나머지는 `public/weather-fx.js`의 렌더링 로직과 `public/style.css`의 키프레임을 다음과 같이 바꾼다:

| 상태 | 현재 문제 | 개선 |
|---|---|---|
| 우박 | 완전한 원이 빗줄기처럼 일정하게 떨어짐 | 살짝 각진 회색 알갱이 + 바닥 근처에서 미세하게 튕기는 키프레임 |
| 눈 | 크기가 균일한 점들이 단조롭게 흔들림 | 크기를 2단계(작음/큼)로 섞고, 큰 눈송이는 더 느리고 크게 흔들리게 |
| 황사/미세먼지 | 개별 입자가 날아다녀 "먼지"보다 "반짝이는 점"처럼 보임 | 입자 수를 줄이고 화면 전체를 덮는 부드러운 그라데이션 안개층을 메인으로, 입자는 보조 요소로만 남김 |
| 안개 | 띠가 3개뿐이라 밴드처럼 보임 | 밴드 5개 + 투명도를 위치별로 다르게 줘서 층layer감 추가 |
| 흐림 | 블롭이 다 같은 속도로 평행 이동 | 블롭마다 크기·속도·투명도를 다르게 줘서 원근감(패럴랙스) |
| 번개 | 화면 전체 밝기 플래시만 있음 | 플래시에 옅은 지그재그 번개 줄기 SVG를 겹쳐서 가끔 보여줌 |
| 폭풍(강풍줄) | 줄무늬 길이·속도·투명도가 균일 | 줄마다 랜덤 값 범위를 넓혀 더 자연스럽게 |
| 햇빛/자외선 아지랑이 | 반복되는 직선 줄무늬라 인공적 | `filter: blur`와 파형 `transform`을 조합한 굴절 느낌으로 교체 |

## 3. 코디 조합표

"오늘의 코디" 카드에 기존 문장형 조언 아래 표를 추가한다. 사용자의 `styleType`(캐주얼/미니멀/스트릿/페미닌/클래식/스포티/빈티지) × 기온대(더움/보통/추움 3그룹으로 단순화) 조합으로 미리 정리한 상의·하의·양말·신발(색상+스타일)을 보여준다. 비/눈이면 신발 설명에 "(방수 소재 추천)"/"(미끄럼 방지 밑창 추천)"을 덧붙인다.

"기타" 스타일(직접 입력)은 조합표 데이터가 없으므로 "직접 입력한 스타일의 조합표는 아직 준비 중이에요"로 안내한다.

이 조합은 실제 SNS(핀터레스트/인스타/X)에서 자주 보이는 스타일별 대표 아이템 구성을 참고해 만든 일반적인 예시이며, 특정 게시물을 그대로 가져온 것은 아니다.

## 4. 추천 스타일 (준비중 자리)

조합표 아래에 "추천 스타일" 섹션을 추가하고, 지금은 정적으로 "곧 실제 코디 이미지를 참고해서 비슷한 스타일을 보여드릴 예정이에요"와 "준비중" 배지만 표시한다. 실제 탭 전환 위젯이 아니라 향후 이미지 연동 시 내용만 갈아끼우면 되는 고정 섹션으로 만든다 (범위 최소화).

## 영향 파일

- `public/app.js`: `tagLabel`/`tipBlockHtml` 시그니처 변경, 헤어카드 배지 숨김 처리, `OUTFIT_TABLE` 데이터 + `buildOutfitTable(profile, w)` + 렌더링 함수 추가
- `public/index.html`: 스킨/메이크업 카드 렌더 호출부는 app.js에서만 바뀌므로 마크업 변경 없음, "오늘의 코디" 카드에 조합표 테이블 + 추천 스타일 섹션 마크업 추가
- `public/weather-fx.js`: 우박/눈/황사·미세먼지/안개/흐림/번개/폭풍 렌더링 함수 조정 (개수·크기·랜덤 범위)
- `public/style.css`: 위 애니메이션들의 키프레임/스타일 조정, 조합표/추천 스타일 섹션 스타일

## 비범위 (Out of scope)

- 실제 핀터레스트/인스타그램 API 연동 (플레이스홀더만)
- "기타" 커스텀 스타일에 대한 조합표 데이터 생성 (준비중 안내로 대체)
- 조합표를 실제 쇼핑몰 제품과 연동하는 것
