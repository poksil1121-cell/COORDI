# 지도 통합 & 도시 검색 정확도 개선 — 설계

날짜: 2026-07-09

## 배경

- 도시 검색(Open-Meteo Geocoding API)이 "안양"처럼 접미사 없는 이름으로 검색하면 실제로 잘 알려진 도시(예: 안양시, 인구 59만)를 놓치고 이름이 같은 무명 마을만 보여준다. 접미사를 정확히 붙여야("안양시") 진짜 도시가 나온다. "광주"도 마찬가지로 "광주시"(경기도, 인구 8만)와 "광주광역시"(인구 140만)를 각각 접미사까지 써야만 찾을 수 있다.
- 선택된 지역이 실제로 어디인지 시각적으로 보여줄 방법이 없고, 다른 지역의 날씨를 둘러볼 방법도 없다.

## 범위

1. 도시 검색 정확도/범위 개선 — 접미사 확장 검색 + 인구 기반 정렬
2. 지도 통합 — 선택된 위치 미리보기 지도(온보딩/대시보드 동일 좌표 재사용) + 클릭해서 들어가는 인터랙티브 탐색 지도(다른 지역 날씨 조회, 그 자리를 새 지역으로 설정 가능)

## 1. 도시 검색 정확도 개선

### 원인

Open-Meteo Geocoding API(GeoNames 기반)는 이름 필드가 정확히 일치해야 매칭된다. "안양시"라는 이름을 가진 레코드는 "안양"으로 검색해도 나오지 않는다. 실제 확인:

| 검색어 | 결과 |
|---|---|
| 안양 | 무명 마을 3개 (population 없음) |
| 안양시 | 안양시 · 경기도 (population 595,644) |
| 광주 | 충남 천안시 소재 무명 마을 1개 |
| 광주시 | 광주시 · 경기도 (population 81,780) |
| 광주광역시 | 광주광역시 (population 1,401,235) |

### 해결 방식

검색어에 흔한 한국 행정구역 접미사(시/군/구/광역시/특별시/특별자치시/특별자치도)를 자동으로 붙인 변형들을 병렬로 검색해서 병합하고, `population` 필드가 있는(=실제 알려진 도시) 결과를 인구수 내림차순으로 우선 정렬한다. 이미 접미사로 끝나는 검색어(예: "안양시")는 변형을 추가하지 않고 그대로 1번만 검색한다.

- 변형 쿼리 중 일부가 실패해도 전체 검색은 계속 진행 (개별 요청에 방어적 처리)
- 병합 시 `이름|admin1|admin2|위도|경도` 조합으로 중복 제거
- 최종 상위 12개까지만 노출

### "구" 단위 세분화는 별도 처리

Open-Meteo/OpenStreetMap 둘 다 한국의 "구"를 독립된 지명으로 인덱싱하지 않아 검색으로는 지원할 수 없다. 대신 아래 지도 기능에서 지도를 직접 클릭해 원하는 지점(구 단위보다 더 정밀함)을 고르는 방식으로 대체한다.

## 2. 지도 통합

### 기술 선택: Leaflet.js + OpenStreetMap 타일

무료, API 키 불필요, CDN으로 로드(이미 폰트를 CDN으로 로드하는 기존 패턴과 동일). OSM 사용 정책상 지도 하단의 저작자 표시(attribution)는 유지한다.

### 2-A. 미리보기 지도 (선택 위치와 100% 동일 좌표)

- 온보딩에서 지역을 확정하면(도시 검색 결과 클릭 또는 "현재 위치 사용" 완료 시) "선택된 지역" 박스 아래에 작은 지도가 나타나고 그 좌표에 마커가 찍힌다.
- 대시보드 날씨 배너 안에도 같은 크기의 작은 지도가 들어간다.
- 두 미리보기 지도는 항상 `profile.region.latitude/longitude` (또는 온보딩 중이면 `selectedCity.latitude/longitude`)를 그대로 사용하므로, 배너에 표시되는 위치·이름과 지도 마커 위치가 구조적으로 항상 일치한다 (별도 좌표 조회 없음).
- 미리보기 지도는 드래그/휠줌/더블클릭줌이 모두 꺼진 정적인 형태이며, 클릭하면 2-B의 탐색 지도가 열린다.

### 2-B. 인터랙티브 탐색 지도

- 미리보기 지도를 클릭하면 화면을 덮는 지도 모달이 열리고, 미리보기와 같은 좌표를 중심으로 확대/이동 가능한 큰 지도가 나타난다.
- 지도의 임의 지점을 클릭하면:
  1. 클릭한 좌표로 기존 날씨 조회 로직(`fetchWeather`/`classifyWeather`)을 그대로 호출
  2. 조회 중에는 클릭한 자리에 "조회 중…" 팝업 표시
  3. 완료되면 그 자리에 위치 이름(역지오코딩) + 현재 온도 + 날씨 상태 + 강수확률 + 자외선 지수를 담은 팝업으로 교체
  4. 팝업 안에 "이 위치로 지역 설정" 버튼이 있어 누르면:
     - 온보딩 중이면 `selectedCity`를 그 좌표로 갱신하고 모달을 닫음 (미리보기 지도도 즉시 갱신)
     - 대시보드에서 열었으면 `profile.region`을 그 좌표로 갱신해 저장하고, 모달을 닫고 대시보드를 새로고침
- 모달은 닫기 버튼 또는 바깥 영역 클릭으로 닫힌다.

### 파일 구성

- `public/map.js` (신규): Leaflet 조작만 담당하는 모듈. `createMiniMap(container, lat, lon, onClick)`과 `openMapExplorer(lat, lon, { onFetchWeatherHtml, onSelectLocation })`, `closeMapExplorer()`를 전역으로 제공. 날씨 조회/렌더링 로직은 몰라도 되고, app.js가 콜백으로 필요한 걸 넘겨준다.
- `public/app.js`: 검색 함수를 변형-검색+정렬 방식으로 교체, 온보딩/대시보드에 미리보기 지도 렌더링 호출 추가, 탐색 지도용 콜백(위치 이름 조회 + 날씨 조회 + HTML 구성, 지역 설정 처리) 구현.
- `public/index.html`: Leaflet CDN 링크/스크립트 추가, 온보딩 지역 확정 박스 아래 미리보기 지도 컨테이너, 대시보드 날씨 배너 안 미리보기 지도 컨테이너, 지도 모달 마크업 추가.
- `public/style.css`: 미리보기 지도, 지도 모달, 팝업 버튼 스타일.

## 비범위 (Out of scope)

- 한국 전체 시/군/구 좌표를 직접 정리한 정적 데이터셋 (좌표 정확성 리스크와 범위가 커서 제외, 지도 클릭으로 대체)
- 실시간 교통/행정경계 오버레이 등 지도 부가 기능
- 해외 지역 검색 정확도 개선 (한국 행정구역 접미사 확장 로직은 한국 지명에 한정)
