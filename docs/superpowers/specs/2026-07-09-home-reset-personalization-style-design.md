# 홈 버튼 리셋 & 개인화(닉네임/나이) & 코디 스타일 — 설계

날짜: 2026-07-09

## 배경

- 우측 상단 ⚙(설정) 버튼은 현재 기존 프로필로 미리 채워진 온보딩 1단계로 이동만 한다.
- 온보딩이 지역/체질/제품 정보만 물어봐서 "내 맞춤"이라는 느낌이 약하다.
- 오늘의 코디 추천이 날씨만 반영하고 사용자의 코디 취향(스타일)은 반영하지 않는다.

## 1. 홈 버튼 (전체 리셋)

- `#settingsBtn`을 `#homeBtn`으로 변경(아이콘·역할이 바뀌므로 식별자도 맞춤), 톱니 아이콘을 집 모양 SVG로 교체, `title`을 "처음부터 다시 시작"으로 변경
- 클릭 시 `window.confirm("정말 처음부터 다시 시작하시겠어요? 기존에 입력한 내용이 모두 사라져요.")`
  - 취소 → 아무 동작 없음
  - 확인 → `localStorage` 삭제, 메모리상 `profile`/`selectedCity` 초기화, `homeBtn.hidden = true`, 빈 온보딩 1단계로 이동(`showOnboarding()` 인자 없이 호출 — 기존에 이미 있는 "빈 폼" 분기를 그대로 재사용)

## 2. 개인화 — 닉네임 · 나이

- 온보�드 1단계 맨 위(성별 위)에 "닉네임"(텍스트, 선택) · "나이"(숫자, 선택) 입력 추가
- 둘 다 필수 아님 — 비워도 "다음" 진행 가능
- `profile.nickname`, `profile.age`로 저장. 완료 요약(4단계)에도 표시
- 닉네임이 있으면 대시보드 상단(날씨 패널 위)에 "OO님, 오늘의 웨더핏이에요" 인사말 표시. 없으면 그 줄 자체를 숨김
- 나이는 이번 범위에서는 저장·요약 표시만 하고 추천 로직에는 반영하지 않음 (범위 확대 시 별도 작업)

## 3. 코디 스타일

- 온보딩 2단계(모발·피부 타입 옆)에 "코디 스타일" select 추가: 캐주얼 · 미니멀 · 스트릿 · 페미닌 · 클래식 · 스포티 · 빈티지 · 기타
- "기타" 선택 시 바로 아래 직접 입력 텍스트 필드 노출 (기존 제품 카테고리 "기타" 입력과 동일한 UI 패턴 재사용)
- `profile.styleType`(key), 기타일 때 `profile.customStyle`(직접 입력 문자열)로 저장
- `buildOutfitRec(profile, w)`로 시그니처 변경(기존 `buildOutfitRec(w)`), 날씨 기반 조언 문장 뒤에 스타일 문장을 이어붙임:

| 스타일 | 추가 문장 |
|---|---|
| casual | 캐주얼하게 편안한 티셔츠나 데님 위주로 매치해보세요. |
| minimal | 미니멀하게 무채색 기본템으로 심플하게 매치해보세요. |
| street | 스트릿하게 오버사이즈 아우터나 스니커즈를 포인트로 매치해보세요. |
| feminine | 페미닌하게 부드러운 소재나 플레어 실루엣으로 매치해보세요. |
| classic | 클래식하게 단정한 셔츠나 슬랙스로 깔끔하게 매치해보세요. |
| sporty | 스포티하게 트레이닝복이나 스니커즈 룩으로 활동적으로 매치해보세요. |
| vintage | 빈티지하게 레트로 무드의 아이템으로 개성 있게 매치해보세요. |
| other | `{customStyle}` 스타일에 맞게 매치해보세요. (직접 입력한 이름을 그대로 문장에 사용) |

## 영향 파일

- `public/index.html`: 홈 버튼 아이콘 교체, 온보딩 1단계에 닉네임/나이 입력, 2단계에 코디 스타일 select + 커스텀 입력, 대시보드에 인사말 요소 추가
- `public/app.js`: `homeBtn` 처리(리셋 로직), `collectProfileFromForm`/`renderSummary`에 닉네임·나이·스타일 반영, `buildOutfitRec` 시그니처 변경 + 스타일 문장 추가, 대시보드 인사말 렌더링
- `public/style.css`: 인사말, 스타일 커스텀 입력 필드 스타일(기존 `.product-custom-category`와 유사)

## 비범위 (Out of scope)

- 나이 기반 추천 로직 (수집·표시만, 로직 반영은 다음 작업)
- 스타일에 따른 헤어/스킨케어 추천 변경 (이번엔 오늘의 코디 카드에만 반영)
- 리셋 확인창을 커스텀 모달로 만드는 것 (네이티브 `confirm()`으로 충분— 파괴적 동작이라 오히려 네이티브 다이얼로그의 낯선 느낌이 신중함을 유도함)
