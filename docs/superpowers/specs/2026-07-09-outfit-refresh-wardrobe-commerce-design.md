# 조합표 새로고침 & 내 의상(사진 분석) & 상품 추천 자리 — 설계

날짜: 2026-07-09

## 배경

- 코디 조합표가 스타일×기온대별로 딱 하나의 조합만 보여줘서 반복해서 봐도 항상 같다.
- 사용자가 실제로 가진 옷으로 조합을 만들어보고 싶어 한다.
- 나중에 실제 상품 구매 연동으로 확장할 자리가 필요하다.

## 1. 조합표 새로고침

각 스타일의 상의(기온대별)/하의/양말/신발 슬롯에 옵션을 2개씩 마련해두고, "🔄 다른 조합 보기" 버튼을 누르면 그 안에서 무작위로 다시 뽑아 보여준다. 완전한 무한 생성은 아니고 스타일별로 꽤 다양한 조합 풀에서 매번 새로 고르는 방식임을 정직하게 유지한다(과장된 "무한대" 문구는 쓰지 않음).

## 2. 내 의상 추가 (사진 분석)

**실현 가능성 확인**: 이 앱은 이미 `server.js`에 선택적 `ANTHROPIC_API_KEY`를 쓰는 `/api/recommend` 엔드포인트가 있고, Claude 모델은 이미지 입력을 지원한다. 같은 패턴으로 실제 동작하는 기능을 만든다 (a "준비중" 표시가 아니라, API 키가 없을 때만 안내 문구가 뜨는 기존 AI 기능과 동일한 처리).

- 대시보드 "오늘의 코디" 카드에 "👕 내 의상 관리" 버튼 → 모달이 열리고 사진 업로드로 의상을 추가/삭제할 수 있음
- 사진은 클라이언트에서 캔버스로 축소(최대 480px)한 뒤 JPEG로 변환해 서버에 전송 (localStorage 용량과 API 전송량을 아끼기 위함)
- 서버는 그 이미지를 Claude에게 보내 `{ category: "top|bottom|socks|shoes|outer", color, style }` JSON으로만 응답받아 그대로 클라이언트에 돌려줌
- API 키가 없으면 기존 `/api/recommend`와 동일하게 400 + 안내 메시지 반환, 클라이언트는 그 메시지를 그대로 보여줌
- 분석된 의상은 `profile.wardrobe` 배열(사진 데이터 URL 포함)에 저장되고, 삭제도 가능
- **조합표 반영**: `buildOutfitTable`이 각 카테고리(상의/하의/양말/신발)마다 먼저 `profile.wardrobe`에 그 카테고리 의상이 있는지 확인하고 있으면 그중 무작위로 사용, 없으면 기존 카탈로그 데이터로 채움 (부분적으로만 등록해도 등록된 부분은 우선 반영됨). 내 옷으로 채워진 항목에는 "내 옷" 배지를 붙여 구분한다.

## 3. 상품 추천 (상업 연동 자리)

실제 이커머스/제휴 API가 없어서 진짜 구매 가능한 상품을 연결할 수 없다. 조합표 아래에 "이 조합과 어울리는 상품 [준비중]" 섹션만 정적으로 추가하고, 나중에 제휴사 API를 붙일 자리로 남긴다.

## 영향 파일

- `server.js`: `/api/analyze-clothing` 엔드포인트 추가, JSON 바디 크기 제한을 이미지 전송을 위해 상향(`express.json({ limit: '5mb' })`)
- `public/app.js`: `OUTFIT_TABLE`을 배열 기반으로 확장, `pickRandom`/`buildOutfitTable`/`renderOutfitTable` 수정, 새로고침 버튼 바인딩, 이미지 리사이즈 헬퍼, 내 의상 모달 열기/닫기/업로드/삭제 로직
- `public/index.html`: 새로고침 버튼, "내 의상 관리" 버튼 + 모달 마크업, "상품 추천 준비중" 섹션
- `public/style.css`: 새로고침 버튼, 내 의상 모달/목록/썸네일/배지, 상품 추천 섹션 스타일

## 비범위 (Out of scope)

- 진짜 "무한" 조합 생성 (옵션 풀을 넓히는 정도로 한정)
- 실제 이커머스 제휴 연동 (플레이스홀더만)
- 사진 여러 벌 동시 인식(한 사진에 옷 하나만 분석)
- 의상 사진의 서버 영구 저장(클라이언트 localStorage에만 저장, 분석 요청 때만 서버를 경유)
